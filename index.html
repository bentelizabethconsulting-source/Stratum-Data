The Workflow
Two phases.
One unified pipeline.
Phase one: extract every keyword from your text corpus, build the document-term matrix, and discover co-occurrence modules. Phase two: anchor those modules in time and fuse them with any structured dataset that shares a timestamp ‚Äî revealing what the text was predicting all along.
üîå
01 ‚Äî INGEST
Connect Corpus
Import from email systems, databases, APIs, or flat files. For emails, the full body text ‚Äî not just subjects ‚Äî is pulled for every message along with sender, timestamp, and thread metadata.
‚úÇÔ∏è
02 ‚Äî EXTRACT
Keyword Extraction
NLP preprocessing: tokenization, stopword removal, lemmatization, and optional n-gram generation. Vocabulary is filtered by document frequency thresholds to control matrix size and noise.
üìä
03 ‚Äî MATRIX
Document-Term Matrix
Documents as rows, extracted keywords as columns. Cell values are TF-IDF weights, raw counts, or binary presence ‚Äî your choice. Stored as a sparse matrix to handle vocabularies of 100K+ terms.
üï∏Ô∏è
04 ‚Äî ANALYZE
Apply Methods
Compute a term co-occurrence network from column-wise correlations. Run Louvain community detection to find keyword modules ‚Äî clusters of terms that appear together across the corpus.
üí°
05 ‚Äî INSIGHT
Visualize & Export
Interactive keyword network graphs, heatmaps, SEM path diagrams, and UMAP projections. Export modules, loadings, and model fit statistics as reports or structured data files.

06 ‚Äî FUSE
Cross-Modal Fusion
Use keyword module timestamps as a join key into any structured dataset. Compute lagged cross-correlations, event-driven windows, and joint SEM models spanning text and numeric data simultaneously.

Example Use Case
JMail.world ‚Äî
every word, analyzed
Instead of searching for keywords one at a time, Stratum extracts every meaningful term from 4,821 email bodies, builds a 4,821 √ó 18,340 document-term matrix, and discovers keyword modules ‚Äî which then become the lens for probing bank transfers, flight logs, and any other timestamped structured dataset.
STEP 01
Ingest full email body text
Stratum connects to JMail.world and pulls the complete body text of all 4,821 messages ‚Äî not just subject lines or metadata. Sender, recipient, timestamp, and thread structure are retained as document-level attributes for downstream filtering and time series analysis.
STEP 02
Extract & weight every keyword
NLP preprocessing tokenizes all 4,821 email bodies, removes stopwords, and lemmatizes tokens. N-grams (e.g. "cash flow", "project deadline") are optionally added. After frequency thresholding, 18,340 terms survive. Each is weighted by TF-IDF: high for terms distinctive to specific emails, low for terms ubiquitous across the whole corpus.
STEP 03
Build the document-term matrix
A sparse 4,821 √ó 18,340 matrix is constructed ‚Äî emails as rows, extracted keywords as columns, TF-IDF weights as cell values. Most cells are zero (most words don't appear in most emails). This sparsity is expected and handled efficiently; it's the structure that makes network analysis possible.
STEP 04
Compute the keyword co-occurrence network
Column-wise correlations across the 18,340 terms produce a term-term co-occurrence matrix. Pairs of keywords that frequently appear in the same emails get high edge weights. Louvain community detection runs on this network and identifies three stable modules: a Financial/Legal cluster, a Project/Operations cluster, and a Compliance/Urgency cluster ‚Äî none of which were defined in advance.
STEP 05
Validate modules with SEM
Each keyword module becomes the indicator set for a latent variable in a Structural Equation Model. CFA tests whether the module's keywords load coherently onto a single factor ‚Äî confirming the cluster reflects a real latent topic, not random co-occurrence. Model fit (CFI=0.94, RMSEA=0.048) validates the three-factor structure across the full corpus.

How it works

Each keyword module produces a module activity time series ‚Äî a curve of how often emails carrying those terms appear across time. That signal is compared against event frequency curves from your structured dataset over the same window.

The window Œ¥t is configurable: search for co-occurrences within minutes, hours, days, or weeks. Lagged correlations find whether text activity leads or follows structured events ‚Äî not just whether they coincide.

When the structured dataset contains named entities ‚Äî merchant names, airline codes, account IDs ‚Äî that also appear as keywords in the email corpus, those records are promoted to direct semantic bridges, strengthening the temporal link.

Analysis Modes
Five ways to
fuse the datasets
Each mode asks a different question about the relationship between keyword module activity and structured event patterns.
üìä
Temporal Overlay
Plot module activity time series alongside event frequency from the structured dataset on the same time axis. Visual alignment reveals whether spikes co-occur without making causal claims.
Do Module A emails and large bank transfers spike on the same days?
‚Üî
Lagged Cross-Correlation
Compute cross-correlation between module activity and event frequency at multiple time lags (‚àí7d to +7d). Identifies whether text activity predicts, follows, or merely coincides with structured events ‚Äî and by how many hours or days.
Module A emails precede anomalous wire transfers by ~4 hours (r = 0.74).
ü™ü
Event-Driven Windowing
For each email tagged to a module, open a configurable time window (e.g. ¬±48h) and extract all structured records within it. Build a secondary co-occurrence matrix: which event types cluster inside Module A windows vs. Module B windows?
Module C emails are followed within 6h by international flight bookings in 38% of cases.
üîó
Shared Entity Linkage
When the structured dataset contains named entities ‚Äî merchant names, airline codes, account numbers ‚Äî that also appear as keywords in the corpus, those records become direct semantic bridges. Temporal and lexical signals reinforce each other.
"Acme Corp" appears in Module A emails and as a transfer recipient on the same dates.
‚ü∂
Joint SEM
Build a Structural Equation Model where both keyword module loadings and structured event patterns are indicators of the same latent construct. Tests whether the two modalities jointly reflect an underlying process ‚Äî and quantifies how much each contributes.
A latent "financial stress" factor loads on Module A keywords AND on wire transfer frequency and amounts.
üö®
Anomaly Alerting
Once a baseline cross-modal co-occurrence pattern is established, flag new records when their temporal relationship to keyword module activity deviates significantly ‚Äî unusual transactions arriving outside typical windows, or module spikes with no corresponding structured event.
Module A spike detected with no corresponding transfer event in the expected ¬±6h window.
